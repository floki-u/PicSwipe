## 1. 项目背景

### 1.1 问题陈述

智能手机用户的照片和视频数量逐年增长，但绝大多数从未被回看过。根据行业数据，平均每位 iPhone 用户存有超过 2,000 张照片，其中大量是截图、重复拍摄、模糊照片和过时的社交媒体图片。这些文件占据了宝贵的设备存储空间，但用户缺乏高效的清理方式——系统相册的多选删除操作繁琐，第三方工具往往过于复杂或过度请求权限。

### 1.2 产品愿景

**让清理照片像刷短视频一样轻松有趣。**

PicSwipe 是一款 iOS 原生工具应用，通过沉浸式的全屏滑动体验帮助用户快速决策哪些照片/视频该删除。核心理念：随机展示、快速决策、批量删除——将枯燥的整理工作变成一个 2 分钟就能完成的轻松任务。

### 1.3 目标与成功指标

| 指标 | MVP 目标 |
|------|----------|
| 单次清理会话完成率 | > 80%（用户开始后完整看完一组） |
| 平均决策速度 | < 2 秒/张 |
| 每次会话删除率 | 20-40%（标记删除的照片占一组的比例） |
| App Store 评分 | ≥ 4.5 |
| 次日留存 | > 30% |

## 2. 用户画像

### 2.1 主要用户：小白（普通大众）

- **年龄：** 18-45 岁
- **特征：** 手机存储经常不足，不太懂技术，希望一键解决问题
- **痛点：** "手机又满了，但一张张看太累了"
- **期望：** 打开就能用，不需要学习，快速释放空间
- **使用频率：** 存储不足时使用，约每月 1-2 次

### 2.2 次要用户：手机重度用户

- **年龄：** 16-35 岁
- **特征：** 大量社交截图、聊天图片、下载图片堆积
- **痛点：** "我有 8000 张截图但只需要其中 100 张"
- **期望：** 能按类型筛选，快速清理特定类别
- **使用频率：** 每周 1 次，形成清理习惯

### 2.3 次要用户：摄影爱好者

- **年龄：** 20-50 岁
- **特征：** 拍照量大，每次拍摄同一场景多张选最佳
- **痛点：** "每次旅行回来有几百张照片需要筛选"
- **期望：** 能全屏查看细节后快速做决定
- **使用频率：** 拍摄活动后集中使用

## 4. 核心用户流程

### 4.1 主流程：快速清理

```
用户打开 App
    │
    ├── 首次使用 ──→ 相册权限请求 ──→ 授权 ──→ 手势教程
    │                                  └──→ 拒绝 ──→ 引导页 ──→ 系统设置
    │
    ├── 选择模式（照片/视频）
    │
    └── 点击「开始清理」
            │
            ▼
        随机抽取一组（N张）
            │
            ▼
        ┌── 滑动浏览 ──┐
        │   上滑=保留    │
        │   左滑=删除    │  ← 循环直到最后一张
        │   下滑=回看    │
        └──────────────┘
            │
            ▼
        确认删除页
        ├── 查看标记的照片
        ├── 可撤回误标记
        └── 点击「确认删除」
            │
            ├── 系统弹窗确认
            │       ├── 确认 ──→ 删除成功 ──→ 结果页
            │       └── 取消 ──→ 回到确认页
            │
            └── 结果页
                ├── 「回到首页」
                └── 「再来一组」──→ 回到滑动浏览
```

### 4.2 筛选清理流程

```
首页 ──→ 点击「筛选后清理」
            │
            ▼
        筛选页
        ├── 选择时间范围（单选）
        ├── 选择相册（多选）
        ├── 查看符合条件数量
        └── 点击「开始筛选清理」
            │
            ▼
        （与主流程相同的滑动浏览 → 确认删除流程）
```

## 5. 页面详细需求

### 5.1 首页（仪表盘）

**页面用途：** App 的入口页面，展示存储状态和清理成就，提供清理入口。

**功能需求：**

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| H-01 | 显示设备存储空间使用情况 | 准确显示已用/总量，误差 < 1GB |
| H-02 | 显示照片和视频各占的空间 | 分别统计并显示 |
| H-03 | 存储进度条颜色随使用率变化 | < 70% 绿色，70-90% 黄色，> 90% 红色 |
| H-04 | 显示累计清理统计 | 显示总清理数量和释放空间，数据持久化 |
| H-05 | 照片/视频模式切换 | 点击切换，选中态有视觉反馈 |
| H-06 | 「开始清理」按钮 | 点击后进入对应模式的滑动浏览页 |
| H-07 | 「筛选后清理」入口 | 点击进入筛选页 |
| H-08 | 设置入口 | 点击进入设置页 |

### 5.2 滑动浏览页

**页面用途：** App 的核心页面，全屏展示照片/视频，用户通过手势快速决策。

**功能需求：**

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| S-01 | 照片全屏展示 | 照片填充屏幕，保持原始比例 |
| S-02 | 上滑保留并下一张 | 照片向上飞出，绿色闪烁反馈 |
| S-03 | 左滑标记删除并下一张 | 照片向左飞出，红色 × 动画 |
| S-04 | 下滑回看上一张 | 上一张从顶部滑入，保留之前的标记状态 |
| S-05 | 右滑无操作 | 轻微弹回，不触发任何动作 |
| S-06 | 手指跟随 | 照片跟随手指移动，有方向倾斜效果 |
| S-07 | 阈值提示 | 滑动超过屏幕 1/3 时显示操作提示文字 |
| S-08 | 未达阈值弹回 | 滑动不足 1/3 松手后弹回原位 |
| S-09 | 顶部状态栏 | 显示返回按钮、已标记删除数、当前进度 |
| S-10 | 底部信息栏 | 显示拍摄日期、文件大小 |
| S-11 | 进度条 | 底部显示当前位置占比 |
| S-12 | 单击切换纯净模式 | 隐藏/显示所有 UI 覆盖层 |
| S-13 | 双指缩放 | 放大查看照片细节 |
| S-14 | 第一张下滑限制 | 弹性反馈，不跳转 |
| S-15 | 最后一张完成后 | 自动跳转确认删除页 |
| S-16 | 视频自动预览 | 静音自动播放前 3 秒，循环 |
| S-17 | 视频播放控制 | 点击暂停/播放，进度条拖动，音量控制 |

### 5.3 确认删除页

**页面用途：** 展示所有标记删除的照片，允许撤回，最终确认删除。

**功能需求：**

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| C-01 | 顶部统计 | 显示标记数量和预计释放空间 |
| C-02 | 网格缩略图 | 展示所有标记删除的照片/视频 |
| C-03 | 撤回标记 | 点击缩略图上的 × 撤回，从列表消失 |
| C-04 | 全屏预览 | 点击缩略图本身可全屏查看 |
| C-05 | 视频时长标签 | 视频缩略图叠加时长 |
| C-06 | 确认删除按钮 | 红色主按钮，显示待删数量 |
| C-07 | 再来一组 | 放弃当前所有标记，开始新会话 |
| C-08 | 全部撤回 | 清空标记，回到浏览页第一张 |
| C-09 | 全部保留处理 | 0 张标记时显示"全部保留"，提供再来一组 |
| C-10 | 全部删除确认 | 全部标记时额外弹窗二次确认 |
| C-11 | 删除结果展示 | 显示清理数量和释放空间 |
| C-12 | 结果页导航 | 提供"回到首页"和"再来一组"两个选择 |

### 5.4 筛选页

**功能需求：**

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| FL-01 | 时间范围单选 | 全部时间/1年前/2年前/3年前/自定义 |
| FL-02 | 自定义日期选择 | 弹出日期选择器，选择起止日期 |
| FL-03 | 相册多选 | 列出所有用户相册和系统智能相册 |
| FL-04 | 相册数量标注 | 每个相册显示照片数量 |
| FL-05 | 实时计数 | 切换筛选条件时实时更新匹配数量 |
| FL-06 | 不足一组提示 | 数量不足时显示提示信息 |
| FL-07 | 重置筛选 | 一键恢复默认筛选条件 |

### 5.5 设置页

**功能需求：**

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| SET-01 | 每组数量选择 | 4 档可选：10/20/30/50 |
| SET-02 | 手势引导重置 | 点击后重新播放教程 |
| SET-03 | 清理历史记录 | 时间线展示历史清理（日期、数量、空间） |

## 6. 非功能需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| App 启动到首页可交互 | < 2 秒 |
| 滑动切换照片响应 | < 100ms（无感知延迟） |
| 图片加载（已缓存） | < 50ms |
| 图片加载（首次） | < 500ms（显示缩略图），全分辨率 < 2s |
| 内存占用 | < 200MB（正常使用） |

### 6.2 兼容性

| 项目 | 要求 |
|------|------|
| 最低系统版本 | iOS 17.0 |
| 设备 | iPhone（不做 iPad 适配） |
| 屏幕 | 适配 iPhone SE 到 iPhone 16 Pro Max |
| 深色模式 | 跟随系统设置 |

### 6.3 安全与隐私

| 项目 | 要求 |
|------|------|
| 数据存储 | 所有数据仅存储在本地，不联网 |
| 权限使用 | 仅请求相册 readWrite 权限 |
| 照片处理 | 不上传、不复制、不缓存照片到 App 沙箱 |
| 删除行为 | 移入系统"最近删除"，30 天内可恢复 |

### 6.4 无障碍

| 项目 | 要求 |
|------|------|
| VoiceOver | 所有页面可用，滑动浏览提供自定义操作替代手势 |
| Dynamic Type | 所有文字适配系统字体大小 |
| Reduce Motion | 开启时简化动画为淡入淡出 |

## 7. 技术方案概要

> 完整技术架构详见设计规格文档。

### 7.1 架构

- **模式：** MVVM（Model-View-ViewModel）
- **UI：** SwiftUI
- **持久化：** SwiftData（清理记录 + 用户设置）
- **相册：** PhotoKit（Photos framework）

### 7.2 关键技术点

| 技术点 | 方案 |
|--------|------|
| 图片加载 | PHCachingImageManager，预取窗口 ±3 张 |
| 视频播放 | AVPlayer + AVPlayerLayer（非 AVPlayerViewController） |
| 文件大小 | PHAssetResource.assetResources → fileSize KVC |
| 手势识别 | SwiftUI DragGesture + 自定义状态机 |
| 删除操作 | PHPhotoLibrary.performChanges + PHAssetChangeRequest.deleteAssets |

### 7.3 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| iOS 系统强制弹出删除确认弹窗 | 用户体验多一步确认 | 在 App 内确认页清楚告知，减少惊讶感 |
| iCloud 优化存储的照片加载慢 | 预览等待时间长 | 先显示低分辨率缩略图，异步加载全图 |
| PHAssetResource fileSize KVC 是非公开 API | 未来 iOS 版本可能失效 | 添加 fallback（返回 0），显示"未知大小" |
| Limited Photos Access 下可访问照片有限 | 用户体验受限 | 显示横幅提示，引导扩大权限 |

## 8. 版本规划

### V1.0 — MVP（预计 4 周）

**目标：** 验证核心清理体验

**范围：**
- 照片模式的完整清理流程（F-01 ~ F-07, F-10 ~ F-13）
- 首页仪表盘基础版
- 设置页（每组数量）

**不包含：** 视频模式、筛选功能、清理历史、无障碍优化

### V1.1 — 功能完善（预计 3 周）

**目标：** 补齐视频和筛选功能

**范围：**
- 视频清理模式（F-08, F-09）
- 筛选页（F-14 ~ F-16）
- Limited Access 处理（F-17）
- 清理历史记录（F-18）

### V1.2 — 体验优化（预计 2 周）

**目标：** 无障碍和边缘情况优化

**范围：**
- VoiceOver 支持（F-19）
- Dynamic Type（F-20）
- Reduce Motion（F-21）
- iCloud 照片处理优化（F-22）

## 10. 附录

### 10.1 术语表

| 术语 | 定义 |
|------|------|
| 清理会话（CleanSession） | 用户一次"开始清理"到"确认删除"的完整过程 |
| 一组 | 一次清理会话中随机抽取的 N 张照片/视频 |
| 标记删除 | 用户在浏览时通过左滑将照片标记为待删除状态 |
| 最近删除 | iOS 系统的回收站，删除的照片保留 30 天 |
