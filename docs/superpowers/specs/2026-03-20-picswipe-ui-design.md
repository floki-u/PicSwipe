> 版本：1.0
> 日期：2026-03-20
> 状态：已确认

## 1. 设计方向

### 1.1 视觉风格

**活力年轻** — 鲜明的渐变色、圆润造型、轻松有趣的氛围，类似 TikTok / Robinhood 的年轻化产品风格。

### 1.2 配色方案：清新薄荷

主品牌色采用翠绿→青绿渐变，传达"清理 = 清爽、释放空间"的核心心智。

| Token | 色值 | 用途 |
|-------|------|------|
| `brand-primary` | `#43e97b` | 品牌主色、主操作按钮起始色 |
| `brand-secondary` | `#38f9d7` | 品牌渐变终止色 |
| `brand-gradient` | `linear-gradient(135deg, #43e97b, #38f9d7)` | 主按钮、品牌元素渐变 |
| `background` | `#111111` | 页面深色背景 |
| `surface` | `rgba(255,255,255,0.06)` ~ `rgba(255,255,255,0.08)` | 卡片/区块背景 |
| `destructive` | `#FF453A` | 删除操作、警告 |
| `destructive-bg` | `rgba(255,69,58,0.15)` ~ `rgba(255,69,58,0.8)` | 删除相关背景 |
| `success` | `#43e97b` | 保留操作反馈（复用品牌主色） |
| `warning` | `#F4C542` | 存储空间警告 |
| `text-primary` | `#FFFFFF` | 主要文字 |
| `text-secondary` | `#888888` | 次要说明文字 |
| `text-muted` | `#555555` ~ `#666666` | 辅助/禁用文字 |

### 1.3 基调

- **深色沉浸式**：全局深色背景（`#111`），让照片和操作元素成为视觉焦点
- **卡片化设计**：所有信息模块以圆角卡片呈现，层次分明
- **极简操作**：核心交互页面 UI 元素尽量少，照片占据绝对主角

### 1.4 圆角规范

| 元素 | 圆角 |
|------|------|
| 手机级容器/Hero 区 | 22-24pt |
| 行动卡片/主要卡片 | 16-20pt |
| 按钮（主操作） | 14-16pt |
| 缩略图/相册封面 | 8-12pt |
| Chip/标签 | 10-20pt（Capsule 风格） |
| 进度条 | 2-4pt |

### 1.5 间距系统

沿用技术 Spec 定义的间距系统，在 UI 原型中的实际应用：

| Token | 值 | 应用场景 |
|-------|-----|---------|
| `xs` | 4pt | 紧凑元素间距（圆点间距、极小 gap） |
| `sm` | 8pt | 卡片内元素间距、网格 gap |
| `md` | 14-16pt | 卡片内 padding、区块间距 |
| `lg` | 20-24pt | Hero 区 padding |
| `xl` | 页面左右边距 | 统一 14-16pt（手机屏幕紧凑） |

## 2. 页面设计方案

### 2.1 首页 — 卡片流型（深色沉浸式）

**设计理念：** 类似 TikTok 的深色沉浸式首页，以卡片流方式组织信息，让用户一打开就感受到"清爽有趣"的氛围。

**布局结构（从上到下）：**

1. **系统状态栏**：标准 iOS 状态栏
2. **品牌 Hero 区**：
   - 居中显示 App emoji（🌿）+ "PicSwipe" 大标题 + 一句话 slogan
   - 背景使用品牌色半透明渐变（`rgba(67,233,123,0.15)` → `rgba(56,249,215,0.08)`）
   - 右上角有径向渐变光晕装饰
   - 圆角 22pt
3. **存储空间胶囊**：
   - 水平排列：文字"92.4 / 128 GB" + 迷你进度条
   - 半透明背景（`rgba(255,255,255,0.1)`），Capsule 造型
   - 进度条颜色随使用率变化（绿 → 黄）
4. **「开始清理」行动卡片**：
   - 品牌渐变背景（最醒目的元素）
   - 左侧 ✨ emoji + 标题"开始清理" + 副文字"随机抽取 20 张照片"
   - 右侧 › 箭头
   - 圆角 18pt
5. **三列数据卡片**：
   - 照片数量（📷 3,842）、视频数量（🎬 286）、已释放空间（🧹 2.3GB）
   - 半透明背景，圆角 14pt
   - 其中照片/视频卡片兼作模式切换：点击切换清理模式，选中态有品牌色边框

**交互行为：**

- 点击「开始清理」→ 进入滑动浏览页
- 点击照片/视频数据卡片 → 切换模式（选中态高亮边框）
- 上滑可显示更多内容（筛选清理入口、设置入口）
- 设置入口通过上滑或导航栏进入

### 2.2 滑动浏览页 — 极简纯净

**设计理念：** 最少的 UI 元素，照片/视频全屏展示成为绝对主角。所有覆盖层保持极度克制，仅提供必要信息。

**布局结构：**

1. **照片/视频层**：完全填充屏幕，保持原始比例（`aspectFill`）
2. **顶部薄状态栏**（半透明，不遮挡照片）：
   - 左：「‹ 返回」文字按钮
   - 中：删除计数标签（`🗑 3`，红色背景 Capsule `rgba(255,59,48,0.85)`，圆角 10pt）
   - 右：进度文字（`5/20`）
   - 字号：12-13px，font-weight 600
   - 无背景蒙层，文字直接叠加在照片上
3. **底部信息区**（居中对齐）：
   - 行 1：拍摄日期（13px，weight 500，opacity 0.9）
   - 行 2：文件大小（11px，opacity 0.6）
4. **底部进度条**：
   - 位于屏幕底部安全区上方
   - 高度 2-3px，背景 `rgba(255,255,255,0.2)`
   - 填充部分使用品牌绿（`#43e97b`）
   - 圆角 1-2pt

**手势交互视觉反馈：**

| 手势 | 视觉反馈 |
|------|----------|
| 上滑（保留） | 照片跟手上移 + 轻微向后倾斜，超过 1/3 屏幕显示绿色"保留 ✓"提示，触发后飞出 + 轻微绿色闪烁 |
| 左滑（删除） | 照片跟手左移 + 向左倾斜，超过 1/3 屏幕显示红色"删除 ×"提示，触发后飞出 + 红色 × 动画 |
| 下滑（回看） | 上一张从顶部滑入，如已标记删除则显示红色半透明删除标记覆盖 |
| 右滑（无操作） | 轻微弹回，不触发任何动作 |
| 单击 | 隐藏/显示所有 UI 覆盖层（纯净模式切换） |
| 双指缩放 | 标准缩放手势查看细节 |

**纯净模式：** 单击后隐藏顶部栏、底部信息和进度条，仅保留照片全屏。再次单击恢复。

**视频模式差异：**
- 进入后自动静音播放前 3 秒循环
- 底部信息区增加时长标签
- 文件大小更醒目显示

### 2.3 确认删除页 — 逐张预览

**设计理念：** 让用户能大图查看每张待删照片的细节，逐张确认后再执行删除。比纯网格更慎重，减少误删。

**布局结构（从上到下）：**

1. **顶部导航栏**：
   - 左：「‹」返回
   - 中：标题"确认删除" + 副标题"左右滑动查看每张照片"（11px，色 #888）
2. **大图预览区**：
   - 高度约 200-220pt，圆角 18-20pt
   - 照片填充，底部渐变蒙层（`rgba(0,0,0,0.7)` → 透明）
   - 蒙层内显示：拍摄日期 + 文件大小/类型
   - 左上角：序号标签（`2/7`，红色背景 `rgba(255,69,58,0.8)`）
   - 右上角：✕ 撤回按钮（圆形，半透明黑色背景 + 毛玻璃）
   - 阴影：`0 6-8px 20-24px rgba(0,0,0,0.4)`
   - 左右滑动切换不同照片
3. **圆点指示器**：
   - 标准 iOS 风格圆点，当前项使用红色长条（`#ff453a`，宽 16px）
   - 其他项为白色半透明小圆点（5-6px）
4. **缩略图导航条**：
   - 水平排列小缩略图（42-48px 方形，圆角 7-8px）
   - 当前选中项有红色边框（`2px solid #ff453a`）
   - 未选中项半透明（opacity 0.5）
5. **释放空间提示**：
   - 居中文字："预计释放 **18.6 MB**"（数字红色加粗）
6. **底部操作区**：
   - 主按钮："确认删除 (7)"，红色背景（`#ff453a`），圆角 14pt
   - 次要操作："再来一组" | "全部撤回"（灰色文字链接）

**交互行为：**

- 左右滑动大图预览区 → 切换照片，联动圆点和缩略图
- 点击缩略图 → 跳转到对应照片
- 点击大图右上 ✕ → 撤回该张标记（从列表移除，计数更新）
- 点击"确认删除" → 系统弹窗二次确认 → 执行删除 → 结果页

**特殊状态：**

- 全部保留（0 张标记）：显示"这组全部保留！🎉"，提供「再来一组」
- 全部撤回后：回到浏览页第一张重新浏览

### 2.4 筛选页 — 可视化滑块

**设计理念：** 用可拖动的时间滑块和可视化的相册封面卡片，让筛选过程直观有趣，而非枯燥的表单。

**布局结构（从上到下）：**

1. **顶部导航栏**：
   - 左：「‹」返回
   - 中：标题"筛选清理"
   - 右："重置"（品牌绿文字）
2. **时间范围区块**：
   - 区块标签："📅 时间范围"（11px，色 #888）
   - 卡片容器（半透明背景，圆角 12pt）：
     - 日期范围显示：起止日期（品牌绿色，14px，weight 600）
     - 可拖动滑块：
       - 轨道高 3-4px，背景 `rgba(255,255,255,0.1)`
       - 填充部分使用品牌渐变
       - 拖动手柄：13-16px 圆形，品牌绿色，深色边框
     - 快速预设按钮：「1年前」「2年前」「3年前」「全部」
       - Capsule 造型，圆角 10-12pt
       - 选中态：品牌绿背景 + 深色文字
       - 未选中态：半透明背景 + 灰色文字
3. **相册选择区块**：
   - 区块标签："📁 相册"
   - 横向滚动的相册封面卡片：
     - 每张卡片宽 68-80px
     - 封面缩略图（方形，圆角 10-12pt，带彩色渐变占位）
     - 相册名（9-10px）
     - 照片数量（8-9px，灰色）
     - 选中态：品牌绿色边框（`2px solid #43e97b`）
     - 支持多选
4. **实时计数卡片**：
   - 居中显示匹配数量（大数字，品牌绿色，18-22px，weight 800）
   - 副文字"张照片符合条件"
   - 半透明品牌色背景 + 边框
5. **底部操作按钮**：
   - "开始筛选清理"，品牌渐变背景

**交互行为：**

- 拖动滑块 → 实时更新日期范围和匹配数量
- 点击预设标签 → 快速定位时间范围
- 点击相册封面 → 多选切换（选中/取消）
- 数量不足一组时 → 计数区显示提示文字

### 2.5 设置页 — 卡片可视化

**设计理念：** 摆脱传统的 iOS 设置列表，用卡片化布局让设置页也保持品牌一致性，关键设置（每组数量）直接操作不用跳转。

**布局结构（从上到下）：**

1. **顶部导航栏**：「‹」返回 + 标题"设置"
2. **品牌卡片**：
   - 品牌渐变半透明背景 + 细边框
   - 左侧 App 图标（40-50px，品牌渐变圆角方形）
   - 右侧 App 名 + 版本号 + 权限状态
3. **每组数量卡片**：
   - 卡片标题："每组数量"
   - 4 个并排选择按钮：10 / **20**（默认选中） / 30 / 50
   - 选中态：品牌渐变背景 + 深色加粗文字
   - 未选中态：半透明背景 + 灰色文字
   - 底部说明文字："每次清理随机抽取的照片数量"
4. **清理历史卡片**：
   - 卡片标题："清理历史"
   - 迷你柱状图（7 列 = 一周，品牌渐变色）
   - 底部可点击"📊 查看完整记录 ›"
5. **其他设置卡片**：
   - 列表行样式（左 emoji + 文字，右 › 箭头）
   - 「👆 重播手势教程」
   - 「🔒 相册权限」
   - 「🌿 隐私政策」

## 5. 深色模式适配

由于整体设计以深色为基调，深色模式下几乎无需改动。浅色模式（跟随系统）的适配策略：

| 元素 | 深色模式 | 浅色模式 |
|------|----------|----------|
| 页面背景 | `#111111` | `#F2F2F7`（systemGroupedBackground） |
| 卡片背景 | `rgba(255,255,255,0.06-0.08)` | `#FFFFFF` + 阴影 |
| 品牌渐变 | 保持不变 | 保持不变 |
| 文字主色 | `#FFFFFF` | `#000000` |
| 文字次要色 | `#888888` | `#666666` |
| 删除红色 | `#FF453A` | `#FF3B30` |

**原则：** 品牌渐变色在两种模式下保持一致，仅调整背景和文字对比度。

## 6. 动画规格

### 6.1 滑动浏览页动画

| 动画 | 参数 |
|------|------|
| 跟手移动 | 1:1 跟随，无延迟 |
| 方向倾斜 | 最大旋转角度 ±10°，随位移线性变化 |
| 阈值判定 | 位移 > 屏幕宽/高的 1/3 |
| 飞出动画 | `spring(response: 0.4, dampingFraction: 0.8)`，飞出距离 = 屏幕尺寸 × 1.5 |
| 弹回动画 | `spring(response: 0.3, dampingFraction: 0.7)` |
| 下一张滑入 | 从对应方向 `opacity: 0 → 1`，`offset → 0`，`spring(response: 0.35)` |
| 绿色闪烁（保留） | 全屏绿色覆盖层 `opacity: 0 → 0.15 → 0`，时长 0.3s |
| 红色 × 动画（删除） | 居中 × 图标放大淡入淡出，时长 0.3s |

### 6.2 Reduce Motion 适配

当 `UIAccessibility.isReduceMotionEnabled` 为 `true` 时：
- 所有飞出/滑入动画替换为简单的 `opacity` 淡入淡出（时长 0.25s）
- 取消方向倾斜效果
- 保留触觉反馈（不受 Reduce Motion 影响）

### 6.3 其他页面动画

| 场景 | 动画 |
|------|------|
| 首页「开始清理」按钮 | 轻微脉冲缩放动画（`scale: 1.0 → 1.02 → 1.0`，循环，时长 2s） |
| 确认删除页大图切换 | 水平滑动过渡，`spring(response: 0.35)` |
| 删除成功动画 | 照片缩小 + 向上位移 + 透明度渐变消失 |
| 进度条更新 | 宽度变化 `animation(.easeInOut(duration: 0.3))` |

## 8. 设计决策记录

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 视觉风格 | 活力年轻 | 契合"像刷短视频一样清理照片"的定位 |
| 配色方案 | 清新薄荷（#43e97b → #38f9d7） | 绿色传达"清理=清爽"，与功能语义一致 |
| 首页布局 | 卡片流型（深色） | 沉浸式体验，与滑动浏览页的深色风格统一 |
| 滑动浏览页 | 极简纯净 | 最少 UI 遮挡，照片为绝对主角 |
