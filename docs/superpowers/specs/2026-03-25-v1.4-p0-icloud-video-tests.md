## 一、iCloud 照片兜底修复

### 问题

`SwipeCardView` 加载 iCloud-only 照片时，如果下载失败或网络差，用户看到的是**永久转圈的 ProgressView**，无法操作也无法跳过。

### 根因

1. `PhotoLibraryService.requestImage()` 的 `isNetworkAccessAllowed = true` 会触发 iCloud 下载，但没有暴露下载进度
2. 下载失败时 `completion(nil)` 被调用，`SwipeCardView` 没有区分"正在加载"和"加载失败"
3. `AssetItem` 没有 iCloud 状态字段

### 设计方案：PHImageManager 进度回调 + 超时计时器

#### 1. PhotoLibraryService 改动

```swift
/// 增强的图片请求方法，支持 iCloud 下载进度回调
func requestImage(
    for asset: PHAsset,
    targetSize: CGSize,
    contentMode: PHImageContentMode = .aspectFill,
    progressHandler: ((Double) -> Void)? = nil,
    completion: @escaping (UIImage?, ImageLoadError?) -> Void
) -> PHImageRequestID
```

- `PHImageRequestOptions.progressHandler` 回调 iCloud 下载进度（0.0~1.0）
- 仅在 iCloud 下载时触发，本地照片不会触发
- completion 新增 `ImageLoadError?` 参数，区分正常 nil 和错误

新增错误枚举：

```swift
enum ImageLoadError {
    case iCloudDownloadFailed
    case timeout
    case unknown
}
```

#### 2. SwipeCardView 三态显示

| 状态 | 触发条件 | UI 展示 |
|------|---------|---------|
| **正常加载** | `image == nil && error == nil && !isDownloadingFromCloud` | 现有 ProgressView() |
| **iCloud 下载中** | `isDownloadingFromCloud == true` | ☁️ 图标 + 进度条 + RPG 文案 |
| **加载失败** | `error != nil` | ❌ 图标 + 错误文案 + 跳过提示 |

**iCloud 下载中 UI：**
- 居中 SF Symbol `icloud.and.arrow.down`（白色，40pt）
- 下方 RPG 风格文案：`"DOWNLOADING FROM CLOUD..."`（Press Start 2P 字体）
- 进度条：使用 DesignSystem 的 `brandPrimary` 色
- 百分比文字：`"65%"`

**加载失败 UI：**
- 居中 SF Symbol `xmark.icloud`（destructiveRed，40pt）
- RPG 文案：`"QUEST FAILED"`（Press Start 2P 字体）
- 副文案：`"SWIPE UP TO SKIP"`（textSecondary 色）

#### 3. 超时机制

- 图片加载启动时同步启动 15 秒 `Task.sleep` 计时器
- 超时后取消 PHImageManager 请求（`cancelImageRequest`）
- 设置 error 状态，显示失败 UI
- 用户可正常上滑跳过（使用现有手势逻辑）

#### 4. 状态管理（SwipeCardView 内部）

新增 `@State` 属性：

```swift
@State private var isDownloadingFromCloud: Bool = false
@State private var downloadProgress: Double = 0.0
@State private var loadError: ImageLoadError? = nil
```

状态流转：

```
loadImage()
├─ progressHandler 首次回调 → isDownloadingFromCloud = true
├─ progressHandler 后续回调 → downloadProgress = value
├─ completion(image, nil) → 正常显示
├─ completion(nil, error) → loadError = error
└─ 15s 超时 → cancelRequest + loadError = .timeout
```

#### 5. 视频模式兼容

视频使用 `requestPlayerItem`，iCloud 视频下载同样可能失败。但 P0 阶段先聚焦照片模式，视频的 iCloud 兜底可以在后续迭代处理（视频本身有 `EnhancedVideoPlayerView` 的 loading 状态）。

## 二、视频模式单测补充

### 问题

`SwipeViewModel` 的视频模式三个核心方法 `advanceToNext()` / `markDeleteAndAdvance()` / `undoLastMark()` 零测试覆盖。

### 方案：在 SwipeViewModelTests 新增视频模式测试区

复用现有 `createTestSession()` helper，新增 `// MARK: - 视频模式操作` 分区。

#### 测试用例

| # | 方法 | 测试名 | 场景 | 预期 |
|---|------|--------|------|------|
| 1 | `advanceToNext` | `test_advanceToNext_movesToNextWithoutMarking` | 正常推进 | index +1, markedForDeletion 不变 |
| 2 | `advanceToNext` | `test_advanceToNext_atLastAsset_setsIsFinished` | 到达最后一张 | isFinished = true |
| 3 | `advanceToNext` | `test_advanceToNext_withNilSession_doesNothing` | session 为 nil | 无状态变化 |
| 4 | `markDeleteAndAdvance` | `test_markDeleteAndAdvance_marksCurrentAndAdvances` | 正常标记推进 | 当前项 marked = true, index +1 |
| 5 | `markDeleteAndAdvance` | `test_markDeleteAndAdvance_atLastAsset_setsIsFinished` | 最后一张标记 | marked = true, isFinished = true |
| 6 | `undoLastMark` | `test_undoLastMark_unmarksLastMarkedItem` | 有 1 个已标记 | 该项 marked = false |
| 7 | `undoLastMark` | `test_undoLastMark_withNoMarkedItems_doesNothing` | 无已标记项 | session 不变 |
| 8 | `undoLastMark` | `test_undoLastMark_withMultipleMarked_onlyUnmarksLast` | 多个已标记 | 只撤回最后一个 |

#### 测试结构

```swift
// MARK: - 视频模式操作

func test_advanceToNext_movesToNextWithoutMarking() {
    // Arrange: session at index 0, 3 assets
    // Act: vm.advanceToNext()
    // Assert: index == 1, assets[0].markedForDeletion == false
}
// ... 以此类推
```
